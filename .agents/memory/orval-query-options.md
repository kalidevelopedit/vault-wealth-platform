---
name: Orval queryKey workaround
description: Generated hooks require queryKey in UseQueryOptions; use `as any` to bypass
---

Orval-generated hooks (e.g. `useListAssets`, `useGetAssetDetail`, `useGetAdminUserDetail`) require `queryKey` in their `UseQueryOptions` type parameter. Passing `{ enabled: boolean }` or `{ refetchInterval: number }` without `queryKey` causes TS2741.

**Why:** Orval generates strict query options types that require `queryKey` to be part of the options object. This is a codegen constraint, not a runtime issue.

**How to apply:** Wrap the query options with `as any`: `{ query: { enabled: !!id, refetchInterval: 30_000 } as any }`. This is safe — queryKey is auto-generated by the hook internally and the `as any` cast only bypasses the TypeScript check, not the runtime behavior.
