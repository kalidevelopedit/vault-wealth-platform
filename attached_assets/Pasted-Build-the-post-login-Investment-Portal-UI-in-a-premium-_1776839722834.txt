Build the post-login Investment Portal UI in a premium ultra-clean dark exchange/dashboard style based on these references.

IMPORTANT:
- Do NOT copy brand logos, trademarks, or exact branded artwork.
- Replicate only the UI language, spacing system, layout behavior, component styling, proportions, hierarchy, and interaction patterns.
- This must feel institutional, high-trust, polished, restrained, and expensive.
- Absolutely no playful startup vibe, no cheap gradients, no bright colored cards, no candy UI, no glassmorphism abuse, no dribbble gimmicks, no oversized shadows, no rounded cartoon styling.
- Ban colored background alerts entirely.
- Ban colorful icon circles, colored tiles, loud badges, and attention-seeking visual noise.
- Any status, alerts, notices, helper states, or warnings must use neutral dark surfaces, subtle borders, and text hierarchy only.
- Icons must be monochrome or near-monochrome.
- Mobile responsive is required.

OVERALL DESIGN DNA:
The interface is a luxury dark trading platform aesthetic:
- full black / near-black foundation
- layered charcoal surfaces
- very subtle navy undertones in some panels
- clean thin dividers
- sharp information density with disciplined spacing
- large high-contrast typography where money/value matters
- soft rounded rectangles, never bubbly
- minimal color usage, reserved mostly for action buttons and market state accents
- strong alignment, strict rhythm, strong edge control
- everything feels engineered, not decorated

COLOR SYSTEM:
Use a restrained dark palette only.

Base backgrounds:
- page background: #050505 to #0A0A0A
- main content surface: #0C0F14
- secondary surface: #11141A
- elevated panel: #151922
- hover surface: #191F28
- border lines: rgba(255,255,255,0.08)
- stronger border for active or focused elements: rgba(255,255,255,0.14)

Text:
- primary text: rgba(255,255,255,0.96)
- secondary text: rgba(255,255,255,0.68)
- muted text: rgba(255,255,255,0.45)
- disabled text: rgba(255,255,255,0.28)

Actions:
- primary action blue: #2563FF or slightly deeper
- hover blue: #2D6BFF
- pressed blue: #1C56E6
- never use multiple accent colors across the base interface

Semantic accents:
- positive numbers: muted emerald, not neon
- negative numbers: muted red, not saturated
- warnings/info: use text + subtle border only, no colored fills

DO NOT:
- no pure bright white backgrounds
- no big glows
- no rainbow highlights
- no colored page sections
- no soft pastel anything
- no frosted transparent cards unless extremely subtle
- no emoji-style visual treatment

LAYOUT STRUCTURE:
Desktop layout should feel like a professional trading portal:

1. Top global navigation bar
- fixed height around 64px to 72px
- full width
- dark black background slightly distinct from page
- very thin bottom border
- left aligned product navigation
- right aligned search, deposit CTA, wallet access, user menu / hamburger
- content vertically centered
- navigation items evenly spaced with tight disciplined gaps
- no oversized paddings

2. Optional left section navigation in account/security/portal views
- fixed width around 220px to 260px desktop
- black sidebar merging with page
- active item shown by subtle filled state, not loud highlight
- each nav item has icon + label
- generous vertical spacing between groups
- item height around 48px to 56px
- rounded rectangle highlight with low-contrast dark fill
- icons monochrome only

3. Main content region
- maximum content width wide, around 1280px to 1440px feel
- large cards stacked vertically with 20px to 28px gaps
- left aligned titles
- data-dense but not cramped
- sections separated by space, not heavy borders

4. Right utility panels where needed
- narrower side cards for tips / FAQs / referral / secondary actions
- same surface language as main panels
- no bright promo styling unless absolutely necessary, and even then keep it extremely subdued

SPACING SYSTEM:
Use an 8px grid.
Core spacing:
- 4, 8, 12, 16, 20, 24, 32, 40
Typical usage:
- card internal padding: 24px desktop, 20px tablet, 16px mobile
- section gap: 24px to 32px
- small control gap: 8px to 12px
- nav item horizontal padding: 14px to 18px
- chip/badge padding: 4px 8px or 6px 10px max
- button height: 40px to 48px
- large hero action button: about 48px to 52px

BORDER RADIUS:
Keep radii restrained and consistent:
- cards: 16px to 20px
- buttons: 999px for pill CTA only when matching reference top bar actions
- regular inputs/dropdowns: 12px to 14px
- nav active backgrounds: 14px to 16px
- small chips/badges: 8px to 10px
Do NOT over-round everything.

TYPOGRAPHY:
Use a modern neutral sans serif with strong clarity.
Recommended feel:
- Inter, SF Pro, Geist, or similar
- slightly tight tracking for headings
- no decorative fonts

Type scale:
- page titles: 44px to 56px, weight 700
- section titles: 28px to 36px, weight 650 to 700
- card titles: 20px to 24px, weight 600
- body: 14px to 16px
- supporting meta: 12px to 13px
- large balance/value numbers: 44px to 56px desktop, 32px to 40px mobile

Typography behavior:
- strong contrast for values
- secondary data beneath in muted text
- dense market rows use tabular numerals
- currency/value tables aligned right
- labels aligned left
- use line-height around 1.15 to 1.25 for headings, 1.4 to 1.55 for body

SURFACE TREATMENT:
Every panel should feel like machined matte metal in digital form.
- background fill mostly solid dark, sometimes with a very slight deep navy gradient
- borders are hairline and soft
- shadows are minimal and mostly ambient:
  box-shadow: 0 8px 24px rgba(0,0,0,0.28)
  or even less
- avoid floating card chaos
- cards should feel anchored and stable

  - compact search field, pill-like, dark inner fill

- nav gap around 20px to 28px

Search field:
- height around 36px to 40px
- pill shape
- dark inset fill
- soft border or inner shadow
- text muted until active
- leading icon monochrome
- no bright outline except focused subtle border

BUTTONS:

Tertiary button / inline actions:

- background #171B22 to #1A1F27

- dropdown shadow should be soft and vertical, never dramatic

CARDS:

A. Overview / balance card
- very large dark panel
- top-left small title

C. Utility side card

D. Settings/security card

MARKET TABLE DESIGN:

- pair symbol bold

INPUTS / SELECTS:
For deposit page and selection controls:
- large dark select fields
- height 48px to 56px
- radius 12px to 14px
- left icon slot, selected label center-left, chevron right
- background slightly brighter than page
- border subtle
- focused state = crisp outline or outer ring using faint white/blue mix
- no colored placeholders

FAQ / TIPS PANELS:
- right column cards
- text uses small body size with strong spacing
- bullets simple, subtle, muted
- links can be slightly brighter blue or primary text
- card padding 20px to 24px
- card radius 16px to 18px

BADGES / CHIPS:
Use sparingly.
- background: slightly lifted dark neutral or ultra-subtle tinted dark blue
- text: small semibold
- no bright neon pills
- no huge corner radius unless tiny chip
- badge height around 20px to 24px
- examples: "New", "Verified", leverage labels, fee labels
- keep them understated and compact

ICONS:
- monochrome or grayscale
- thin to medium stroke
- crisp 1.5px to 2px feel
- no colorful icon backgrounds
- if an icon container is needed, make it very dark neutral with subtle border
- remove decorative icon glows

ALERTS / NOTICES RULE:
This is critical.
Any alerts, warnings, confirmations, helper banners, or notices must NOT use:
- green background
- red background
- yellow background
- blue filled background
- colorful side strips
- bright icon circles

Instead:
- use dark card background
- subtle border
- small monochrome icon
- title + body text
- maybe one understated text link
- if severity must be indicated, use text tone or a tiny colored dot only, but ideally keep even that minimal

ANIMATION / MOTION SYSTEM:
Motion must feel premium and nearly invisible.
No flashy motion.

Allowed animations:
- hover background fade: 120ms to 180ms ease-out
- dropdown open: opacity + translateY(4px to 8px), 160ms to 220ms cubic-bezier(0.2, 0.8, 0.2, 1)
- card hover: maybe 1px translateY max or none at all
- focus ring fade: 120ms
- sidebar active shift: color/background transition only
- table row hover: background and text contrast transition only

Do NOT use:
- bounce
- springy overshoot
- scale popping
- large blur transitions
- parallax
- shimmer loaders unless extremely subtle skeletons
- glow pulses
- animated gradients

Microinteractions:
- buttons: slight brightness increase on hover
- menus: smooth fade and anchored movement
- selected nav item: subtle underline or background fill transition
- tabs: crisp active state, no pill carnival
- number changes in market widgets can flash slightly with muted semantic text color, but never with bright background

SHAPE LANGUAGE:
- predominant shape is rounded rectangle
- circles only for tiny avatar/icon holders if needed
- pills for main CTA and search
- no weird angled corners
- no blob shapes
- no skeuomorphic nonsense
- geometric discipline everywhere
